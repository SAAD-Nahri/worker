# Phase 4.7: Media And Asset Layer

## Objective

Add a controlled media and visual-asset layer after the connected text system is proven.

This phase exists because the current system is mostly text-complete, but blog publishing and Facebook distribution still lack a full visual handling baseline.

## Why This Phase Matters

Media is not optional polish for this business.

For a Facebook-to-blog model, visuals influence:

1. post attractiveness,
2. perceived content quality,
3. brand consistency,
4. click behavior,
5. publish completeness.

If media handling stays undefined, the system remains incomplete even if the text chain is solid.

## Main Responsibilities

1. define allowed media source types,
2. define rights-safe media handling rules,
3. define asset records and linkage to blog and social outputs,
4. define media brief generation from approved draft context,
5. define operator review rules for media before publishing,
6. define minimum alt-text and asset-metadata expectations.

## Required Outputs

1. media and visual-asset policy,
2. media brief contract,
3. media record and linkage model,
4. WordPress-featured-image and Facebook-asset handling baseline,
5. operator review path for selected assets.

## What This Phase Must Not Do

1. scrape copyrighted images casually,
2. treat random web images as reusable by default,
3. make video production the first media scope,
4. turn the repo into a design-system project,
5. bypass human review for visual assets.

## Candidate Media Sources

The system may later support a controlled mix of:

1. operator-owned uploaded assets,
2. licensed stock assets,
3. AI-generated assets,
4. simple branded graphic templates,
5. carefully reviewed source-linked visuals only when rights and policy are clear.

## Risks

1. rights and licensing mistakes,
2. low-quality visuals that weaken the post,
3. visual inconsistency across blog and Facebook,
4. adding media automation before the asset policy is defined,
5. letting media scope become broader than the business actually needs.

## Definition Of Done

Phase 4.7 is done when:

1. one approved draft can receive a reviewed media package,
2. the asset can be linked cleanly to blog and Facebook outputs,
3. media provenance and review state are visible,
4. the system has a rights-safe baseline for visual publishing,
5. the repo can move into later runtime/deployment work, and then Decision Layer work, with a more complete content artifact.
