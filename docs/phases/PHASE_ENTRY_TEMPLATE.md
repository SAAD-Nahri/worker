## Entry Checklist

### 1. Upstream contract check

- [ ] Upstream phase closeout has been reviewed.
- [ ] The upstream contract is stable enough to build against.
- [ ] The authoritative upstream inputs are known.

### 2. Phase boundary check

- [ ] The team agrees what this phase owns.
- [ ] The team agrees what this phase does not own.
- [ ] Out-of-scope work has not been mixed into the start plan.

### 3. Design readiness check

- [ ] The first implementation target is defined.
- [ ] The first required record shapes or contracts are defined.
- [ ] The first required quality gates are defined.

### 4. Operational readiness check

- [ ] The relevant docs are current.
- [ ] `TODO.md` reflects the active execution work.
- [ ] The first validation path for this phase is understood.
