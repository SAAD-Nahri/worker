# Intake Log Template

## Purpose

This template is the working intake log for the pilot.

Use it to record each reviewed source item before it becomes a draft candidate. The goal is to preserve source judgment, not just final publishing outcomes.

## Recommended Columns

| review_date | source_id | source_family | source_url | raw_title | topic_fit | template_fit | facebook_hook_fit | dedupe_risk | decision | reviewer_notes |
|---|---|---|---|---|---|---|---|---|---|---|
| YYYY-MM-DD | `src_example` | `food_editorial` | `https://...` | Example title | strong / medium / weak | food_fact / curiosity / reject | strong / medium / weak | low / medium / high | advance / hold / reject | short practical note |

## Column Guidance

### `topic_fit`

Suggested values:

1. `strong`
2. `medium`
3. `weak`

### `template_fit`

Suggested values:

1. `food_fact_article`
2. `food_benefit_article`
3. `curiosity_article`
4. `reject`

### `facebook_hook_fit`

Suggested values:

1. `strong`
2. `medium`
3. `weak`

### `dedupe_risk`

Suggested values:

1. `low`
2. `medium`
3. `high`

### `decision`

Suggested values:

1. `advance`
2. `hold`
3. `reject`

## Daily Rule

Every reviewed item should be logged once. If an item is not worth logging, it probably is not worth reviewing.

## Weekly Use

At the end of each week, this log should help answer:

1. which sources produced the best candidates,
2. which sources produced mostly rejects,
3. which template families appeared most usable,
4. which sources should be downgraded or promoted.
