# Week 1 Intake Plan

## Purpose

This document defines the exact first-week operating plan for the manual-first pilot.

The goal is to make week one disciplined, realistic, and useful. We are not trying to maximize content volume in week one. We are trying to validate the source mix and build a strong first draft queue.

## Week-One Goal

By the end of week one, the system should have:

1. sampled the active source registry,
2. reviewed a controlled number of candidate items,
3. selected a strong first batch for drafting,
4. identified weak sources or weak item patterns early.

## Operational Targets

### Intake targets

1. review `18-24` source items,
2. shortlist `10-12`,
3. advance `6-8` to drafting.

### Source mix targets

1. `food_editorial`: `11-14` reviewed items,
2. `ingredient_reference`: `3-4` reviewed items,
3. `curiosity_editorial`: `2-3` reviewed items.

This keeps the first batch grounded in quality while still testing click-oriented curiosity patterns.

## Day Structure

Week one should run as a 5-day focused intake cycle.

### Day 1: Feed Health And First Screening

Goals:

1. check all active feeds,
2. review `4-5` candidate items,
3. reject obvious poor-fit material quickly,
4. confirm the registry is realistic for daily use.

Priority sources:

1. Tasting Table,
2. Food Republic,
3. Curious Cuisiniere.

### Day 2: Primary Source Sampling

Goals:

1. review `4-5` more items,
2. start the first real shortlist,
3. check whether primary sources are producing consistent quality.

Priority sources:

1. Tasting Table,
2. Food Republic,
3. Saveur.

### Day 3: Diversity Check

Goals:

1. review `3-5` items,
2. test ingredient-reference and selective-curiosity fit,
3. compare quality versus click potential.

Priority sources:

1. Curious Cuisiniere,
2. Saveur,
3. Mashed.

### Day 4: Selective Editorial Sampling

Goals:

1. review `3-5` items,
2. test Bon Appetit selectively,
3. decide which sources are actually feeding the first draft queue.

Priority sources:

1. Bon Appetit,
2. Tasting Table,
3. Food Republic.

### Day 5: Queue Lock

Goals:

1. review the shortlist,
2. compare duplicate or angle-collision risk,
3. lock `6-8` items for the drafting queue,
4. downgrade any clearly weak source.

## Item Advancement Rule

A source item should advance only if all of the following are true:

1. it clearly fits the food-facts niche,
2. it maps to one of the approved blog templates,
3. it has a usable Facebook hook angle,
4. it is not an obvious duplicate or weak angle collision,
5. it feels worth spending a publish slot on.

## Source Retention Rule

For week one:

1. keep a source active if it produces at least one strong candidate in its first review batch,
2. downgrade a source if it produces zero strong candidates after `4` reviewed items,
3. pause a source if it produces mostly recipe-only, gossip-like, or low-value material.

## Review Cadence

The review process should be:

1. source feed scan,
2. quick item triage,
3. checklist review,
4. advance, hold, or reject,
5. end-of-day shortlist cleanup.

Do not let "hold" become a dumping ground. Held items should either be reconsidered quickly or dropped.

## Week-One Deliverables

By the end of week one, produce:

1. a reviewed item list,
2. a shortlist,
3. a final draft queue of `6-8` items,
4. a source downgrade or keep decision for each active source,
5. notes on what item patterns looked strongest.

## What Not To Do In Week One

Do not:

1. expand the source set unless a source clearly fails,
2. chase volume at the expense of quality,
3. force weak items into the queue,
4. compensate for weak sources by increasing AI involvement,
5. treat curiosity-only signals as more important than transformation quality.

## Professional Standard For Week One

Week one is successful if the result is a clean draft queue, not a large pile of mediocre candidates.

The correct outcome is:

1. fewer stronger items,
2. better source judgment,
3. clearer template fit,
4. a more confident Phase 1 start.
