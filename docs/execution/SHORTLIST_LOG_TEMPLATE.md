# Shortlist Log Template

## Purpose

This template is the working shortlist log for items that survive intake and are approved for drafting consideration.

Use it to preserve the bridge between source review and the drafting queue.

## Recommended Columns

| shortlist_date | source_id | source_url | raw_title | selected_template | shortlist_priority | key_angle | draft_status | queue_intent | notes |
|---|---|---|---|---|---|---|---|---|---|
| YYYY-MM-DD | `src_example` | `https://...` | Example title | `food_fact_article` | high / medium / low | short angle description | pending / drafted / rejected | week_1 / week_2 / hold | short practical note |

## Column Guidance

### `selected_template`

Use one of:

1. `food_fact_article`
2. `food_benefit_article`
3. `curiosity_article`

### `shortlist_priority`

Suggested values:

1. `high`
2. `medium`
3. `low`

### `draft_status`

Suggested values:

1. `pending`
2. `drafted`
3. `rejected`

### `queue_intent`

Suggested values:

1. `week_1`
2. `week_2`
3. `hold`

## Practical Use

The shortlist should stay small and intentional. If it grows too large, it means intake filtering is too weak.

## Weekly Review Questions

At the end of each week, use this log to answer:

1. which shortlisted items actually turned into good drafts,
2. which source families convert best into draftable content,
3. which template type gets chosen most often,
4. which shortlisted items should be dropped instead of carried forward.
