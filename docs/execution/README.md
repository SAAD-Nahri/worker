## How These Docs Fit

1. `MASTER_PLAN.md` defines the long-term direction.
2. `docs/specs` defines the structural rules.
3. `docs/execution` defines how we start professionally without overbuilding.

## Rule

These docs are meant to create a serious first operating cycle, not a fake launch.

The first goal is to prove:

1. the sources are good enough,
2. the formatting system is usable,
3. the review loop is manageable,
4. the blog-to-Facebook workflow can run repeatedly,
5. the data lineage is preserved from day one.

Now that the first Source Engine slice exists and the Content Engine baseline is active, this folder also defines how to operate those phases without skipping ahead into later phases. It also records live acceptance evidence instead of relying only on planned workflows.

Approval automation is intentionally not treated as an execution default here. That remains a later gated capability after Decision Layer work and shadow-mode validation are complete.
