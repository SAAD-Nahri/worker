# Media And Visual Asset Policy V1

## Purpose

This document defines the future baseline for handling media once the connected text system is proven.

## Core Rule

Media must be rights-safe, reviewable, and linked to the publish chain.

The project must not treat images as an afterthought or as a scrape-first shortcut.

## Required Future Baseline

The later media phase should define:

1. allowed asset sources,
2. review state for assets,
3. asset linkage to blog and social outputs,
4. minimum metadata such as provenance and alt text,
5. safe operator workflow for selecting or generating assets.

## Allowed Direction

The system may later support:

1. operator-owned assets,
2. licensed assets,
3. approved AI-generated assets,
4. simple branded templates or overlays.

## Disallowed Direction

The system must avoid:

1. casual reuse of copyrighted source images,
2. untracked visual assets,
3. auto-publishing unreviewed images,
4. growing into a broad creative-automation platform before the business needs it.

## Definition Of Done

This policy is satisfied when later media implementation has a safe baseline for:

1. provenance,
2. review,
3. linkage,
4. publish usability.
